# PRD: Expand Note

> We're going to build a multi-phase set of features to support the use case for expanding the contents of a note (or article, we should discuss naming later - note vs article vs ?). Assume we're on a "Note page". In the first phase we will create a modal chat which should just be the main chat reused. We will add a button beside the "Note" and "Close" buttons that says "Expand" (or possibly "Generate"). When you click that button a modal will appear with a chat interface. It should have the note that we're looking at as context. In phase two, we will add a new button to the chat interface while in modal mode - Overwrite Note (or perhaps Merge with Note). When you click the Overwrite Note button the text from the chat message will replace the note in the main note view. Here's an example workflow: I have selected a note from the library on Tim Berners-Lee in the left sidebar and am viewing it (it's a poor note with only one line of text); I click the "Expand" button beside the Note button. Then a modal appears. The modal allows me to provide additional direction, I type "write an article about Tim Berners-Lee"; the original note and the user's message are sent to the LLM and the response is displayed in the modal window; I click the Overwrite Note button on the message from the LLM (the message with the response from the LLM); my note is replaced with the contents of the LLM response. Finally, in phase three we will make notes full pages and remove the close button (we want to be able to fluidly move in and out of chat); to aid the user we will add a "Recently Viewed Notes" bar (it shouldn't be labeled) below the header. Every time you view a new note the note should go into the Recently Viewed Notes bar (perhaps the last 20 pages, rotating them out as we go), the titles should be displayed in the bar and when you click a title you will go to that page. Finally we will add back and forward buttons to help a user navigate back through the list of recently viewed pages.
